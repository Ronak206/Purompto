# Purompto - AI Prompt Engineering SaaS

An AI-powered prompt generation platform that helps users create perfect prompts through intelligent clarifying questions. Built with modern technologies for a professional, ChatGPT-like experience.

![Purompto](https://img.shields.io/badge/Next.js-16-black?style=flat-square&logo=next.js)
![TypeScript](https://img.shields.io/badge/TypeScript-5.0-blue?style=flat-square&logo=typescript)
![MongoDB](https://img.shields.io/badge/MongoDB-Atlas-green?style=flat-square&logo=mongodb)

## ✨ Features

### 🤖 AI-Powered Prompt Engineering
- **Intelligent Questions** - AI asks clarifying questions to understand your exact needs
- **Streaming Responses** - Watch prompts generate in real-time
- **Perfect Results** - Get optimized prompts that work first time

### 💬 ChatGPT-like Experience
- **Real-time Chat History** - Every message saved instantly
- **Session Recovery** - Resume any conversation from sidebar
- **Grouped History** - Chats organized by Today, Yesterday, This Week
- **Collapsible Sidebar** - Maximize your workspace

### 🎨 Modern UI/UX
- **Dark/Light Theme** - Toggle between themes
- **Responsive Design** - Works on desktop and mobile
- **Smooth Animations** - Professional feel with loading states
- **Copy to Clipboard** - One-click copy for prompts
- **shadcn/ui Components** - Avatar, Badge, Button, Input, Sidebar components

### 👨‍💼 Admin Panel
- **User Management** - Create, update, delete users
- **Access Control** - Enable/disable user accounts
- **Notes System** - Track payment status, notes per user
- **Prompt History** - View all prompts generated by users

## 🚀 Quick Start

### 1. Clone & Install
```bash
git clone https://github.com/Ronak206/Purompto.git
cd Purompto
npm install
```

### 2. Environment Setup
Create `.env` file:
```env
# Database
MONGODB_URI=mongodb+srv://user:pass@cluster.mongodb.net/purompto

# Authentication
JWT_SECRET=your-random-jwt-secret-min-32-chars
ADMIN_SECRET=your-admin-panel-password

# AI API
OPENROUTER_API_KEY=your-openrouter-api-key
OPENROUTER_API_KEY_2=backup-api-key-optional
OPENROUTER_API_KEY_1=another-backup-optional

# App URL
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### 3. Run Development Server
```bash
npm run dev
```

### 4. Access the App
- **Home**: http://localhost:3000
- **Admin Panel**: http://localhost:3000/admin

## 🗄️ Database Models

### User
```typescript
{
  email: string;              // Unique email
  password: string;           // Hashed with bcrypt
  name?: string;              // Display name
  isActive: boolean;          // Account status
  totalPromptsGenerated: number;
}
```

### Chat
```typescript
{
  chatId: string;             // Unique chat ID
  userId: ObjectId;           // Owner reference
  title: string;              // Chat title
  messages: [{                // Conversation history
    id: string;
    role: 'user' | 'assistant';
    content: string;
    questions?: string[];
    createdAt: Date;
  }];
  status: 'active' | 'completed';
  result?: string;            // Final generated prompt
}
```

## 🔌 API Reference

### Authentication
```http
POST /api/auth/login      # Login
POST /api/auth/logout     # Logout
GET  /api/auth/me         # Current user
```

### Chat Operations
```http
GET  /api/chat            # List all chats
GET  /api/chat?chatId=x   # Get specific chat
POST /api/chat            # Create/update chat
DELETE /api/chat?chatId=x # Delete chat
```

### AI Operations
```http
POST /api/analyze         # Generate questions
POST /api/generate        # Generate prompt (streaming)
```

### Admin Operations
```http
GET  /api/admin/users           # List users
POST /api/admin/users           # Create user
PUT  /api/admin/users           # Update user
DELETE /api/admin/users?userId=x # Delete user
```

## 🎯 Business Model

Purompto is designed as a **managed SaaS service**:

1. **Admin creates accounts** for clients after receiving payment
2. **Clients login** and use the service - no payment processing on platform
3. **Admin manages access** - can enable/disable accounts

### Workflow
```
Client pays → Admin creates account → Client uses service
     ↑                                        │
     └──────── Admin controls access ←────────┘
```

- **Password Hashing**: bcrypt with 12 rounds
- **JWT Tokens**: Secure, http-only cookies
- **Request Signing**: Timestamp + nonce validation
- **Rate Limiting**: Ready for production
- **Input Sanitization**: XSS prevention
